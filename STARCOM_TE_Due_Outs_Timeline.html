🔵 PHASE 1 — PRE-TEST (Before Test Execution Begins)
Timeline Due Out / Action Owner / Responsible Party Type Guide Ref Notes / Caveats
~18 months before test STARCOM Awareness & Involvement Determination
STARCOM must be aware of the new program and make an involvement determination. Failure to do so within 18 months of planned test start triggers an exception to standard funding through program elements.
 STARCOM S3C Mandatory Pg. 62 Funding PE exceptions apply if awareness occurs within 18 months of test start.
≥ 180 days before first OT activity Test Concept Brief
Summarizes the campaign plan to STARCOM, DEL 12, and TES leadership. Must be completed NLT 180 days before the start of the first operational test activity for programs on the DOT&E oversight list.
 ITF / Test Director (TD) DOT&E Oversight Pg. 123 Approval level based on risk, cost (ACAT), and special interest of the program.
Throughout Early Involvement / Campaign Planning Funding Agreement with Program Office
A funding agreement between STARCOM and the program office must be reached before test planning efforts beyond Early Involvement can commence.
 STARCOM FM / SPO Mandatory Pg. 62 Per U.S. Code Title 10 and DoDI 5000.98, program office funds 100% of program-specific test costs.
Early Involvement / Campaign Planning Phase ITT Charter (Integrated Test Team)
Formal, signed charter describing team membership, responsibilities, resources, and products. Includes event-driven deliverables table, conflict resolution chart, and DRB/TDSB/JRMET charters.
 Program Office (owns); TES (supports) Mandatory Pg. 64 Required per DoDI 5000.98. Reviewed every 12 months or after each major decision review.
Early Involvement Phase TEMP / T&E Strategy (Initial / Updates)
Overarching living document detailing test conduct for the lifecycle of the program. Inputs from LDTO and OTA required. Must be updated at each Milestone for MDAP programs.
 SPO (develops); STARCOM ITF (inputs) Mandatory Pg. 64–65 DOT&E provides inputs for oversight programs. Contains required IDSK section.
Early Involvement Phase Integrated Decision Support Key (IDSK)
Required part of TEMP/T&E Strategy for all DoD programs. Maps operational capabilities and technical capabilities to decisions and data sources. Initial ICT meeting typically in Early Involvement phase.
 IDSK Core Team (ICT): PM, Ops Decision-makers, LDTO, OTA Mandatory Pg. 65–67 Required for all DoD programs per DoDM 5000.100.
Campaign Planning Phase Test Campaign Plan (TCP)
Documents TEMP/T&E Strategy updates, test schedule, data sources, test events, M&S needs, T&E risk, and resources (personnel, assets, facilities, ranges, threats, etc.).
 ITF / Test Director (TD) Mandatory Pg. 120 Updates TEMP/T&E Strategy significantly during Campaign Planning phase.
4–6 weeks before Test Design Workshop (TDW) I Battlespace Development (Four Tasks)
Complete four tasks: (1) Mission Analysis, (2) Develop COIs, (3) Assign Combat Operations to each COI, (4) Decompose each Combat Operation into Combat Operational Threads (COTs).
 Test Team Mandatory Pg. 76 Must be completed before TDW I can start.
Campaign / Test Planning Phase M&S Accreditation Plan
Establishes benchmark for all V&V activities. Must identify timeline of VV&A activities relative to TRR and test start/end. Submitted to S3C Technical Advisor (TA) for approval.
 Test Director (TD); S3C TA (approves) If M&S Used Pg. 133 Required per DoDI 5000.61 when M&S is used to support test decisions.
Prior to TRR (Goal: before TRR) M&S Accreditation Report
Ensures STARCOM and DOT&E leadership are aware of all risks associated with using M&S. If not complete by TRR, TD must brief M&S risks and expected publication date at TRR.
 Test Director (TD) Recommended Pg. 157 Per MIL-STD-3022 and DoDI 5000.61. Risk brief required at TRR if not complete.
Test Planning Phase Test Plan (and supporting documentation)
Reviewed by Technical Review Board (TRB) chaired by Technical Review Authority (TRA). Assesses test requirements, techniques, approaches, and objectives per DAFI 99-103.
 Test Director (TD); TRB (reviews) Mandatory Pg. 147 Required per DAFI 99-103 for all tests.
Test Planning Phase Test Safety Plan
Documents hazards, safety risks, and mitigations. Includes Baseline Safety Report (BSR), Test Hazard Assessments (THAs), required waivers/deviations, and mishap reporting responsibilities.
 Test Team / TES Safety Office Mandatory Pg. 147 Reviewed by Safety Review Board (SRB). Required for all tests. Details in DEL12I 91-220.
Per DEL12I 91-220 (risk-based schedule) Technical Review Board (TRB)
Independent technical review of the Test Plan. Chaired by Technical Review Authority (TRA). Minimum required per DAFI 99-103 for all tests.
 TRA (chairs); Test Team (briefs) Mandatory Pg. 147 Scheduling per DEL12I 91-220 based on technical complexity and risk.
Per DEL12I 91-220 (minimum days based on risk level) Safety Review Board (SRB)
Required for tests with Low, Medium, or High safety risk level. NRR (Negligible Risk Review) used for Negligible safety risk tests. Scheduling timeline based on risk level per DEL12I 91-220.
 SRB Chair; Test Team; DEL12/SE Mandatory Pg. 149 Output: Safety Review Board Summary Memorandum. Specific day requirements in DEL12I 91-220.
Prior to TEA Approval (before TRR) Test Execution Authority (TEA) Approval of Test Package
TEA (DEL12/CC or ITF Director) accepts the technical and safety risk. TEA approval of the test package is required before going to TRR. If SUT is owned/operated by another org, their commander must concur first.
 TEA (DEL12/CC or ITF Director) Mandatory Pg. 150 TEA may request Test Approval Briefing (TAB) before final approval. Required before any dry runs that direct action on SUT.
Recommended: Prior to TRR Dry Runs (Test Procedures & Data Tools)
Dry runs of test procedures should occur prior to TRR. Data management and analysis tools must be dry-run before test starts. Results discussed at TRR.
 Test Team Recommended Pg. 150–152 TEA must approve test team before dry runs that direct actions on SUT. Issues documented; procedures updated before execution.
≥ 15 days before OT&E start PEO Certification Memo (MOTRC)
Program Executive Officer (PEO) or delegated official certifies the system ready to enter dedicated OT&E and sends certification memo to STARCOM. IAW DAFMAN 63-119.
 PEO / Program Office Mandatory Pg. 152 Not required for OA or EOA. Test team and S3C prepare acknowledgement memo in response. Both memos must define SUT scope and responsibilities.
Prior to TRR Certification Acknowledgement Memo (MOTRC)
Prepared by test team and S3C in response to the PEO Certification Memo. Sent to: PEO, DAF/TE, S3C, and DOT&E (oversight programs only). Addresses readiness, SUT scope, and responsibilities.
 Test Team & S3C Mandatory Pg. 152, 155 IOT&E programs also produce a "Readiness for Operational Test Certification Acknowledgement Memo" at TRR.
Prior to test execution Test Readiness Review (TRR)
Formal briefing by TD to TRR Chair and key stakeholders. Conducted prior to test team deployment. Must address: purpose of test, test overview, SUT description, test limitations, readiness recommendations from test team/SPO/operational units.
 Test Director (TD); TRR Chair (per DEL12I 99-112) Mandatory Pg. 155 Chair determined by DEL12I 99-112 based on risk, oversight status, and ACAT level. External stakeholders invited only with TRR Chair approval.
≥ 30 days before test Risk Management Readiness Review (RMRR)
CFC/S4S process to manage risk when using an operationally accepted weapon system for non-operational purposes (including testing). Due to S4S/S33 NLT 30 days prior to test.
 Test Team (inputs); CFC counterparts (develop/staff) If Operational Asset Used Pg. 157 Required any time an assigned weapon system is used for testing, demos, experiments, studies, or advanced training. No official format — typically a white paper or PPT.
Prior to test execution Authority to Connect (ATC)
Required when connecting two systems with different Authorizing Officials (e.g., cyber test toolkit and ground system). Signed by the AO of the system under test. Test team is responsible for securing ATC.
 Test Team; Authorizing Official (AO) of SUT If Connecting Different-AO Systems Pg. 158 Not required if both systems are authorized by the same AO and have an ISA in eMASS records.
Prior to test execution (track throughout) Authority to Operate (ATO) / Interim Authority to Test (IATT)
RMF process and artifacts managed by Program Office; approved by Space AO (CFC S-Staff). TD should track ATO/IATT status proactively — delays have caused test execution delays.
 Program Office (manages); TD (tracks) Mandatory Pg. 158 Test team supports ATO/RMF process through cybersecurity tests but responsibility rests with Program Office.
Before first test point Program Office Safety Release Letter
PM provides a safety release for the system prior to each developmental and operational test involving personnel. Identifies known ESOH hazards and their risk acceptance.
 Program Manager (PM) Mandatory Pg. 150 Required for tests involving personnel.
Before first test point (prior to TRR is recommended) Test Start Memo
Notifies T&E stakeholders that test execution has been approved via TRR and that ITF has committed resources per the approved test plan. Sent prior to execution of first test point.
 Test Director (TD) Mandatory Pg. 168 Distribution list agreed to at TRR per DEL12I 99-112.
🔴 PHASE 2 — DURING TEST EXECUTION
Timeline Due Out / Action Owner / Responsible Party Type Guide Ref Notes / Caveats
Within 24 hours of discovery CAT I Deficiency Report (DR) Notification
All suspected CAT I DRs must be reported to the Test Director and Program Office within 24 hours of discovery. CAT I Safety DRs also require immediate notification to DEL12 Safety Office (SE) and often result in test pause/stop.
 Test Team; Test Director (TD) Mandatory Pg. 178 Per T.O. 00-35D-54. CAT I Safety DRs may trigger a pause/stop test action.
Within 72 hours of discovery CAT II Deficiency Report (DR) Notification
CAT II DRs require notification to the appropriate authority within 72 hours of discovery. Documented IAW T.O. 00-35D-54 and ITF guidance.
 Test Team; Test Director (TD) Mandatory Pg. 178 DRB charter designates who reviews, validates, prioritizes, and submits identified deficiencies.
Ongoing during test Test Execution and Status Reports (TEASRs)
Periodic status reports distributed to all stakeholders per the data distribution plan. ITF Director approves all TEASRs prior to external release.
 Test Director (TD); ITF Director (approves) Mandatory Pg. 174 Must not violate OPSEC/classification guidance. Frequency per program data distribution plan.
As required (per test plan) Test Pause Memo
Sent when test execution is paused. Establishes re-entry criteria. Triggers TRAC review and may require re-certification depending on scope of changes.
 Test Director (TD) As Required Pg. 168–172 TRR not required to restart after pause unless directed by ITF Director, DEL12/CC, or STARCOM/CC.
As required (per test plan) Test Stop & Decertification Memo
Sent when test is stopped and system decertified (e.g., critical safety or security issue). TRR is required before test can restart after a stop.
 Test Director (TD) As Required Pg. 170–172 TRR required to restart after a stop test.
🟢 PHASE 3 — POST-TEST (After Test Execution Ends)
Timeline Due Out / Action Owner / Responsible Party Type Guide Ref Notes / Caveats
NLT 24 hours after test complete Test Complete Memo
Sent when all planned test points are complete and active testing has ceased. Informs operational community that personnel may resume normal operations. Summarizes test execution start/end, units involved, test points, and estimated LTE date.
 Test Director (TD) Mandatory Pg. 172–173 Must NOT include T&E results, analysis, conclusions, or recommendations. If LTE occurs before test complete, memos are combined.
NLT 24 hours after Last Test Event (LTE) End of Test / LTE Memo
Sent when: all test objectives assessed, data validated by TDSB, data available to all government organizations, last JRMET/TDSB event complete, and data validated for analysis. Triggers the final report coordination timeline clock.
 Test Director (TD) Mandatory Pg. 173 LTE starts the clock for test reporting and releases test resources (people, ops floors, systems) back to owners.
After LTE — per agreed timeline Final Test Report (OT&E / Assessment Report)
Full report with Executive Summary, Test Description, Test Design and Results, Test Resources Used, and Recommendations. Must follow DEL 12 Style Guide and templates. Date communicated in LTE memo.
 Test Director (TD); ITF Director (signs) Mandatory Pg. 195, 214 Draft reports not circulated outside STARCOM. ITF Director may verbally share observations before signing. Specific timeline per program/DEL12 guidance.
After LTE — per agreed timeline Final Results Briefing
Briefing of final report findings to leadership and stakeholders. Date estimated and communicated in both Test Complete Memo and LTE Memo.
 Test Director (TD); ITF Director Mandatory Pg. 173, 216 Coordinate with DOT&E action officer for oversight programs. Key guidance: "No surprises." Pre-coordinate responses to anticipated questions.
Within 30 days after final report approval or closeout decision Test Program Closeout Checklist & MFR
TES reviews closeout checklist, identifies applicable items, works POC actions, and prepares MFR. Staffed for HQ coordination (S1, S3C, S4/6, S5/8/9, HO). Sent to S3C to close program on SharePoint.
 TES / Test Director (TD) Mandatory Pg. 219 Directed in program's Mission Tasking Order (MTO). Templates on DEL 12 SharePoint.
Within 30 days after closeout decision Funds Closeout Actions
Cancel due-outs (if no longer required), terminate contracts/task orders, reconcile MIPRs/Project Orders, submit receiving reports, coordinate with STARCOM/FM on redistribution.
 TD / OL Chief / TES/CC; Resource Manager (POC) Mandatory Pg. 220 Includes AF Forms 616, contracts, supply/equipment accounts, and MIPRs.
After final report approval Lessons Learned Capture
TD must work with STARCOM S9 to ensure applicable lessons learned are captured by the DEL 10 Lessons Learned team.
 Test Director (TD); STARCOM S9; DEL 10 Mandatory Pg. 219 Part of test closeout phase. DEL 10 Lessons Learned team manages the database.
🟣 ONGOING / PERIODIC REQUIREMENTS (Throughout the Test Lifecycle)
Frequency / Trigger Due Out / Action Owner / Responsible Party Type Guide Ref Notes / Caveats
Every 12 months or after each major decision review ITT Charter Review & Update
ITT leadership reviews and updates charter to ensure testing is integrated and membership reflects skills needed for current phase of the program.
 Program Office; ITT Leadership Mandatory Pg. 64 Required per DoDI 5000.98.
At each Milestone (MDAP); as needed (other programs) TEMP / T&E Strategy Update
Must be updated at each subsequent acquisition or program decision, or following major program changes (funding, acquisition, capability updates, milestone decisions).
 SPO (develops); STARCOM ITF / DOT&E (inputs) Mandatory Pg. 65, 120 Mandatory updates at each Milestone for MDAP per DODM 5000.100.
Throughout all phases Deficiency Report (DR) Management
Active DRs from previous tests or integrated tests should be reviewed and if appropriate converted to DRs prior to the start of OT&E. DR status reviewed at working-level prior to formal DRB meetings.
 Test Team; DRB; Action Point (designated by PM) Mandatory Pg. 175–180 IAW T.O. 00-35D-54. DRB charter defines review, validation, prioritization, and submission responsibilities.
Prior to each test phase / program increment Test Resource Plan (TRP) Updates
Documents test resource requirements and should be updated throughout the program lifecycle. Used to inform POM submissions for both STARCOM and the program office.
 Test Team; STARCOM FM Mandatory Pg. 62, 67 Documented in TEMP/T&E Strategy. Enables early identification of long-lead test resources.
