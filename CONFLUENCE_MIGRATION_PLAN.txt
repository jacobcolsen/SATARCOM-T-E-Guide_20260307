================================================================================
STARCOM T&E TRACKER — CONFLUENCE MIGRATION PLAN
Phase-by-Phase Build & Test Guide
================================================================================

OVERVIEW
--------
This plan migrates the STARCOM T&E Program Tracker from a standalone HTML app
into Confluence. Each phase is written as a prompt you can hand directly to
Claude to build that phase, then test it before moving on.

Confluence capabilities used across phases:
  - Page hierarchy (Space > Parent > Child pages)
  - Page Properties macro + Page Properties Report macro (portfolio views)
  - Status macro (colored status badges)
  - Task List macro (checkboxes)
  - Panel macro (colored callout boxes)
  - Table markup (task tables)
  - HTML macro (custom widgets, progress bars) — requires admin to enable
  - Page Templates (reusable program page skeleton)
  - Labels/tags (filtering by program phase)
  - Confluence Cloud vs Server/DC: note where features differ

NOTE ON HTML MACRO: Phases 4+ benefit greatly from the HTML macro being
enabled in your Confluence instance. Ask your admin. Without it, those phases
fall back to native macros only (still functional, less polished).

RECOMMENDED TEST ORDER: Complete and verify each phase before starting the next.

PHASE 1 — SPACE SETUP & PAGE HIERARCHY
================================================================================
WHAT YOU ARE BUILDING:
  The Confluence space structure that mirrors the app's navigation:
  executive view (portfolio), individual program pages, and reference pages.

PROMPT TO USE:
  "I am building a STARCOM T&E Program Tracker in Confluence. Help me set up
   the page hierarchy. I need:

   1. A root/home page called 'STARCOM T&E Program Tracker' with a brief

   2. The following child pages under the root:
      - '📋 Program Portfolio' (will become the executive view)
      - '📁 Programs' (parent page; individual programs go here as children)
      - '📚 T&E Task Library' (reference: all 40 tasks with descriptions)

   3. Label conventions I should use:
      - Labels on program pages: te-program, phase-pre, phase-during,
        phase-post, phase-ongoing, status-active, status-complete,
        status-archived
      - How to apply labels in Confluence and where they appear

   Give me the exact Confluence markup (wiki markup or instructions for the
   visual editor) to create each page with placeholder content so I can
   verify the hierarchy is correct before adding real content."

TEST AFTER PHASE 1:

  - Confirm the breadcrumb trail reads:
    Space > STARCOM T&E Program Tracker > [child page]
  - Confirm you can navigate between pages.
  - Add label 'te-program' to a test child page under Programs and verify

     LTE Date            | [date]
     Current Phase       | [Status macro — Pre-Test / During Test / Post-Test

     DT Complete         | [text, e.g. '0/14']
     Program Lead        | [user mention field]
     Last Updated        | [date]

   SECTION 2 — Program Highlights
   A Panel macro (yellow/warning style) titled '📌 Program Highlights for
   Leadership' with a placeholder text area for the program update narrative.

     ID | Phase | Task Name | Owner | Type | Status | Due Date | Risk | Notes

APPENDIX B — KNOWN LIMITATIONS & WORKAROUNDS
================================================================================

1. NO AUTOMATIC DUE DATE CALCULATION
   The HTML app calculates due dates from Test Start/TRR/LTE inputs.
   Confluence has no formula macros for date arithmetic.
   WORKAROUND: Provide a separate 'Date Calculator' prompt (ask Claude to
   calculate all 40 due dates given your 3 milestone dates, then paste the
   results into the Due Date column of your program page).

2. NO AUTOMATIC STATUS ROLL-UP
   The HTML app auto-derives 'At Risk' and 'Overdue' from dates.
   WORKAROUND: Weekly maintainer check (Phase 9 checklist).
   ALTERNATIVE: If you use Jira, create Jira issues for each task and link
   them to the Confluence page — Jira can auto-flag overdue issues.

3. HTML MACRO REQUIRES ADMIN ENABLEMENT
   Not all Confluence instances allow arbitrary HTML execution.
   WORKAROUND: Use colored Status macros and manual text counts if HTML
   macro is unavailable.

4. PAGE PROPERTIES REPORT HAS COLUMN LIMITS
   Very wide tables may not display well on all screen sizes.
   WORKAROUND: Limit Portfolio columns to the 6 most important fields.

5. MULTI-USER CONCURRENT EDITING
   Confluence allows only one editor at a time per page (edit lock).
   WORKAROUND: Program Leads own their own program page; Portfolio page
   is edited only by the space admin.
